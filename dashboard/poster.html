TRACK A · 01–05 Signal chain Audio → AST
01intake

Audio intake

Any container in, normalised signal out. Live microphone or recorded file.

Acceptwav · mp3 · mp4 · m4a · ogg · flac · live
Output16 kHz mono PCM
02window

Sliding window

One-second analysis frames at 0.5 s hop — 50 % overlap, so transients never fall on a window boundary.

Frame1.000 s
Hop0.500 s · 50 % overlap

Log-mel spectrogram

04classifier · binary

Audio Spectrogram Transformer — drone yes / no

86 M-parameter backbone, 12 transformer layers (multi-head attention + MLP). The 527-class AudioSet head is amputated and replaced with a binary linear projection 768 → 2.

HeadLinear 768 → 2 · returns p(drone)
Params86 M · 12 layers

TRACK B · 06–07 Intelligence layer Type & novelty
06type · prototype matching

Drone-type classification — prototype matching

The 768-dim AST hidden state is cosine-matched against precomputed embedding prototypes. Each prototype is the mean of a small reference library. Currently classifies Shahed-136; new types added by dropping in new clips, no retraining of the trunk.

Shahed-136 0.974 · τ 0.965
Methodcosine similarity vs prototype · rolling 4-s mean
Threshold0.965 · validated on 70 shahed + 330 other-drone windows
Library5 reference clips · 38 s · ≈ 1 % disk vs LoRA adapter
Extensionadd prototypes for DJI · Bayraktar · fixed-wing — no retraining
07open-set

Open-set novelty flag

When p(drone) sits in the ambiguous band — above the detection threshold but below 0.55 — for several consecutive windows, the source is flagged as an unfamiliar UAV signature.

Triggerp(drone) ∈ [τ, 0.55] · ≥ 3 of last 5 windows
Output"Unknown UAV — possible threat"
TRACK C · 08–09 Spatial layer
multi-microphone TDoA & coherence
08triangulation · GCC-PHAT

GCC-PHAT TDoA + least-squares localiser — with position lock

Time-difference-of-arrival per microphone pair → linear least-squares for (x, y, z). Solutions with high residual or implausible distance are rejected. The map shows a marker only once three consecutive accepted positions agree within 2 m — until then, the dashboard reads "locating…" rather than dancing.

InputsN-channel audio · mic positions in metres
MethodGCC-PHAT τ̂ · linear LSQ → ℝ³ · residual ≤ 1.0 m
Lock3 consecutive solutions within 2 m before broadcast
Smoothmedian-of-5 + 50/50 EMA
09decoy check

Decoy check — spatial coherence

Sliding 1-second triangulations across the last 4 seconds form a cloud of recent positions. Three statistics on that cloud decide whether the source is moving (real), stationary (decoy), or incoherent (noise / multipath).

Decoystd < 0.5 m AND smoothness < 0.5 m
Noisesmoothness > 8 m AND not linear
Defaultreal_drone

11operator dashboard

Bilingual operator dashboard

Real-time WebSocket stream drives every panel: threat indicator with drone-type chip, p(drone) gauge, bearing & range readout, event log, microphone editor with live VU bars, cueing JSON viewer, source bar with device picker and threshold slider. Arabic / English toggle. Dark and light themes. Logo and palette sampled from the Saamid mark; Thmanyah typography.

12VU bars

Per-microphone audio levels

Per-channel RMS computed every 0.5 s. Surfaces in the dashboard as live VU bars beside each microphone slot. Channels falling below the silence floor are flagged so the operator catches a dead cable instantly.

Cadence0.5 s · per channel RMS
Alertsilence floor — dead cable
13hot-swap source

Hot-swap audio source

Active source — live device, replay of an uploaded recording, or no source — switches at runtime without restart. Per-microphone upload modal accepts one synchronized recording per mic and stacks them into a multi-channel buffer.

Sourcesdevice · upload · none
Switchruntime · no restart
14live threshold

Live-tunable detection threshold

Operator slider in the dashboard pushes the detection floor to the backend in real time (debounced 150 ms). Lets the operator compensate for ambient noise without restart.

Range0.10 – 0.60
Debounce150 ms · live to backend
15rate & channel adapt

Sample-rate & channel-count adaptation

Captures at the audio interface's native sample rate (typically 48 kHz on pro USB devices) and resamples to 16 kHz internally. When the input is single-channel, triangulation is skipped honestly rather than fabricated.
